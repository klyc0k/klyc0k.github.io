Mobile CapTOP

Mobile adaptive web version of the DDOT CapTOP (Capitol Traffic Operation Platform) system. This platform provides DDOT traffic managers and operators to access the data from global network, while other CapTOP applications are restricted in the internal network. Although it is currently under development, the app will incorporate the data and streams from most of DDOT managed field devices, including: Dynamic Message Signs, Traffic Signals, Roadside/Tunnel Cameras, Incidents, Speed Sensors, Count Stations, and other new infrastructures.

Based on the Twitter Bootstrap framework, the web app is responsive and fitted into mobile devices. Our team is focusing on providing management functions to DDOT users. At the next phase, we will upgrade the scalability and make partial of the services public available, and hope to benefit massive D.C. residents and commuters.

  • Date: 2014 - present

  • Development Tools:
    JavaScript, bootstrap, jquery, C#, .NET Framework, Apache,
    Python, Flask, SQL Server, AWS, GoogleMaps API

Open CapTOP Viewer

The Open CapTOP Viewer was developed back in 2012. At the time that web-based application becoming popular, the purpose of this platform was to provide a more convenient environment for operators and managers to access the data. Unlike our previous windows based CapTOP application, this web-based platform does not require installation on individual computers.

On this platform, we developed more management-wise functions. For example, the Automatic Vehicle Location (AVL) system, provides the capability of tracking and monitoring the patrol vehicles, and helps the manager to allocate patrols to be on scene. Other functions such as the operator performance statistics, and incident report evaluation, are for managers to evaluate the performance of the patrollers and operators.

Not only for managers. We refactored most of functions of the windows based CapTOP program onto this web-based platform. DDOT employees (operators, signal planners, engineers, technicians) can easily access and view partial of data by their authorized accounts.

  • Date: 2012 - present

  • Development Tools:
    C#, ASP.NET, JavaScript, jquery, MS IIS, MS SQL Server, GoogleMaps API, HighCharts

Traffic Data Visualization

DDOT collects significant quantities of traffic and incident data. In this project, we aimed on developing a tool to visualize the data. The visualization shows the spatio-temporal evolution of the transportation system in addition to using macroscopic fundamental diagrams to display the network status.

Several interactive visualization schemes based on d3.js and HighCharts were developed to visualize the traffic data. These charts show the spatio-temporal trend and the hierarchical relations of the intersections and the corridors. Partial of development code is available at https://github.com/ddot-toa/Traffic_data_Viz.

  • Date: 2014 - present

  • Development Tools:
    JavaScript, jquery, d3.js, HighCharts, Python, Flask, MS SQL Server, MS IIS

Tweet Labeling Tool

This is a side product I developed for the research group to view and label civil unrest related tweets. We were trying to build a SVM-based classifier to identify civil unrest related tweets(in Spanish!). The purpose of this tool was to collect more labeled data. By recognizing the civil unrest tweets, we are able to infer the occurrence and location of civil unrest events automatically by content-based geocoding and deep analysis models. A preliminary filter model built by our labeled data is available at https://github.com/klyc0k/EDSFilter.

  • Date: 2013 - 2014

  • Development Tools:
    C#, ASP.NET, JavaScript, jquery, MS IIS, SQL Server, GoogleMaps API

Incident Reporting App

The app was designed for DDOT patrollers to report incidents. Incorporates with the built-in GPS and camera, patrollers can easily report incidents with scene pictures and a precise location. Besides, voice memo is provided in case patrollers are not able to input the details with the keyboard. This app runs on iOS devices and was developed in Objective-C.

  • Date: 2012

  • Development Tools:
    Objective-C, MS-IIS, SQL Server, GoogleMaps API

CapTOP Windows Application

Capital Traffic Operation Platform (CapTOP) was initially deployed in DDOT Traffic Management Center (TMC) in 2007 to support traffic operations and incident management. Since then, CapTOP has been playing a critical role in achieving efficient traffic operations and incident management in the district.

After Virginia Tech cooperated with the DDOT ITS team, we started to integrate more data source into the platform, and develop functions to utilize the data.

Not only for viewing data, we also developed functions to control the devices. The figure shows the dynamic message sign setting function.

To make CapTOP to be a even better integrated operation and monitoring platform, we explored the system architecture of DDOT infrastructure, and find the way to improve it.

We collected integrated data and video stream from variant devices, including dynamic message signs, roadside/tunnel cameras, traffic signals, count stations, multiple speed sensors sources, and others.

To provide capability on device extensions, some side tools were developed for users without programming knowledge to manage our system. For example, the figure shows a tool for implementing a signal display scheme for a newly constructed intersection.

  • Date: 2010

  • Development Tools:
    C#, .NET Framework, ArcGIS, SQL Server

Miscellaneous

Two programs (which are not confidential) are shared on my GitHub repos, including a data retrieval program, and a serial port dynamic message sign control library.

The figure shows the GUI of data retrieval program for collecting travel time data from InRix. This program automatically retrieves data in XML format from InRix in a user defined cycle, parses the data, and stores into DDOT ITS database. The source code of this retriever is available at this link

The figure shows an application that applies the serial DMS control library. This program automatically estimates the travel time of specific highway sections, and display the travel time on the message signs. The serial DMS control library is available at the link. It includes a serialSNMP class that would be helpful for people who are trying to communicate with an old field device that is connected via RS232 serial port.
